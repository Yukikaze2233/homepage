# Homepage 架构说明

## 项目定位
这是一个基于原生 `HTML + CSS + JavaScript` 构建的静态个人主页项目。整个站点不依赖前端框架，也没有构建流程，核心特点是结构简单、资源清晰、易于直接修改。

---

## 整体架构

### 1. 页面入口层
- `index.html` 是整个网站的唯一入口文件。
- 页面内容主要分为两大区域：
  - 左侧信息区：头像、地点、时间线
  - 右侧主内容区：欢迎语、描述、社交入口、项目卡片、技能展示
- 文案、链接、卡片结构等核心内容都直接维护在 `index.html` 中。

### 2. 样式层
- `static/css/style.css`
  - 负责页面布局
  - 卡片样式
  - 动画效果
  - 响应式适配
  - 移动端显示规则

- `static/css/root.css`
  - 负责主题变量
  - 背景图配置
  - 渐变色配置
  - 卡片透明度与全局遮罩透明度

这种拆分方式把“结构样式”和“主题参数”分开，后续调整视觉风格时更容易维护。

### 3. 交互层
- `static/js/script.js` 负责：
  - 明暗主题切换
  - 图片弹窗预览
  - 卡片按压反馈
  - 页面加载过渡
  - 控制台欢迎信息输出

交互逻辑较轻，主要用于增强体验，不参与页面内容生成。

### 4. 资源层
- `static/img/`
  - 存放头像、背景图、项目卡片图片、网站图标、弹窗图片等位图资源

- `static/svg/`
  - 存放 SVG 资源，例如技能图、装饰图等

- `static/fonts/`
  - 存放本地字体文件，用于标题和页面文字渲染

### 5. 部署层
- `Dockerfile`
  - 将整个静态站点打包进 Caddy 容器镜像

- `docker-compose.yaml`
  - 用于启动 Caddy 服务容器

- `Caddyfile`
  - 定义域名、静态资源根目录以及文件服务规则

这意味着项目既可以走纯静态托管，也可以走基于 Caddy 的容器部署。
